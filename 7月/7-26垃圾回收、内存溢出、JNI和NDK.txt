        异常分为Error和Exception
        Error:内存溢出、线程死锁等系统问题
        Exception:系统异常和普通异常
            系统异常：软件本身导致的，如NullPointException等
                        不强制处理 所以称为unchecked异常
            普通异常：运行环境导致的，如网络断线，硬盘空间不够等
                        必须try...catch或throws，所以也称为checked异常


    内存溢出和内存泄漏的区别
        参考：http://blog.csdn.net/buutterfly/article/details/6617375
        内存溢出 out of memory
            你要求分配的内存超出了系统能给你的，系统不能满足需求，于是产生溢出
            比如申请了一个integer,但给它存了long才能存下的数，那就是内存溢出
        内存泄露 memory leak
            程序在申请内存后，无法释放已申请的内存空间，一次内存泄露危害可以忽略，但内存泄露堆积后果很严重，无论多少内存,迟早会被占光

    运行时异常与一般异常有何异同?
        运行异常：虚拟机的通常操作可能遇到的异常，用户不必处理
        一般异常：是用户可以抛出的异常，如果抛出了一定要处理

    Java 语言中的 volatile 变量可以被看作是一种 “程度较轻的 synchronized”；与 synchronized 块相比，volatile 变量所需的编码较少，并且运行时开销也较少，但是它所能实现的功能也仅是 synchronized 的一部分。

    Array和ArrayList的不同点：
     Array可以包含基本类型和对象类型，ArrayList只能包含对象类型。 Array大小是固定的，             ArrayList的大小是动态变化的。 ArrayList提供了更多的方法和特性，比如：
        addAll()，removeAll()，iterator()等等。 
    对于基本类型数据，集合使用自动装箱来减少编码工作量。但是，当处理固定大小的基本数据类型的时候，这种方式相对比较慢

    Heap和stack有什么区别?
        heap:堆 
            使用new创建的对象都放在堆里，不随方法的结束而消失，而方法中的局部变量使用final修饰后，放在堆中，而不是栈中。
        stack：栈
            程序进入一个方法时会自动分配空间给它，当方法结束时，内存也就释放

    java的垃圾回收机制,开发人员如何使用它
        它是在不可预知的情况下对内存堆中已经死亡的或长时间没有使用的对象进行回收
        开发人员不能时时调用垃圾回收器对某个对象进行回收，因为JAVA语言规范并不保证GC一定会执行。

    MVC的各个部分都有那些技术来实现?如何实现?
         MVC是Model－View－Controller的简写。
         "Model" 代表的是应用的业务逻辑（通过JavaBean，EJB组件实现），
          "View" 是应用的表示面（由JSP页面产生），
          "Controller" 是提供应用的处理过程控制（一般是一个Servlet），
          通过这种设计模型把应用逻辑，处理过程和显示逻辑分成不同的组件实现。
          这些组件可以进行交互和重用。
